# エンタイトルメントは Phase 1 で構造だけ作り、強制は Phase 3 の課金と同時に ON

Free/Pro のエンタイトルメント判定について、Phase 1 では「各機能の入口を単一のゲートに通す**構造**」のみを実装し、**サイト数上限（Free 3件）や Pro 機能ロックの実際の強制（enforcement）は行わない**。強制は Phase 3 で Stripe 課金・ライセンス検証が有効化されるのと同時にフラグで ON にする。

## 背景

元仕様の Phase 計画は、Phase 1 に「サイト数上限の適用」を含めつつ、Stripe 課金・Pro 有効化を Phase 3 に置いていた。この通りだと Phase 1〜2 の間は「Pro になる手段が存在しないのに Free 上限だけが効く」状態になり、初期採用者が4サイト目で詰む。

## 決定

- Phase 1: ゲートの構造（単一の入口）を作り込むが、上限・機能ロックは**インアクティブ**。全機能アンロックで提供。
- Phase 3: 課金有効化と同時に enforcement を ON。
- **初期ユーザー（課金開始前の利用者）は無償 Pro として据え置く（grandfather）**。課金開始時の Free への降格は反感を生むため採らない。

## Consequences

- 「サイト数上限の適用」は「適用の仕組みをフラグ一つで効く状態にする」と読み替える。
- grandfather 対象の判定基準（いつまでの利用者か）は Phase 3 の課金設計時に確定する。
