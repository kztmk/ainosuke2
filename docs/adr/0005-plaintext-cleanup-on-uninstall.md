# アンインストール時の config 平文残留への対処（OS で非対称）

接続中のままアプリをアンインストール/削除すると、`claude_desktop_config.json` に平文の認証情報が残留し、本アプリの「平文露出を最小化する」セキュリティ物語が破綻する。これを OS ごとに非対称な手段で緩和する。

- **Windows**: NSIS アンインストーラにフックを入れ、本アプリが書き込んだエントリ（`env.WP_MCP_MANAGER_ID` で識別、[ADR-0001](./0001-site-identity-in-claude-config.md)）を確認のうえ削除する。
- **macOS**: .app のゴミ箱削除にはフックを掛けられないため自動消去は不可能。設定画面に「すべて接続解除してアンインストール準備」操作を用意し、ヘルプに「アンインストール前に全接続解除を」を明記。起動時の「接続中のまま長期間放置」警告（24h 警告）が事実上の保険となる。

## 背景

セキュリティ設計が「接続解除時にアプリが平文を消す」ことに依存しているが、アンインストールという最も普通の終わり方がそのフローを迂回する。macOS と Windows でアンインストールのフック可否が異なるため、単一の解決策では塞げない。

## Consequences

- macOS では平文残留を完全には自動防止できないことを既知の制約として受容し、運用（ボタン＋ヘルプ＋警告）で緩和する。
- 削除対象の特定は表示名ではなく `env.WP_MCP_MANAGER_ID` に依存する。
