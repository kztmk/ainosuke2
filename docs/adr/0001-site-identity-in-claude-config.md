# claude_desktop_config.json 内でのサイト識別キー戦略

`mcpServers` のキーには、安定した内部 `id`（uuid-v4）ではなく**サイトの表示名**を使う。理由は、このキーが Claude Desktop 上でユーザーに見えるサイト識別名になるため、人間可読性を優先したから。

トレードオフとして表示名は可変なので、接続中サイトの改名は「旧キー削除＋新キー追加」として扱い、保存時に一意性（アプリ内 ＋ `claude_desktop_config.json` 既存キー全体）を強制する。

自社が書き込んだエントリの識別は表示名に依存させず、各エントリの `env` 内に隠しマーカー `WP_MCP_MANAGER_ID: <uuid>` を埋めて行う。`env` を選んだのは、エントリ直下に未知キーを置くと Claude Desktop のスキーマ検証で弾かれうるのに対し、`env` は自由な文字列マップで子プロセスに環境変数として渡るだけで安全に無視されるため。

## Consequences

- 改名・削除・「他アプリの MCP 設定を保持する」操作は、表示名ではなく `env` 内マーカーで自社エントリを特定して行う。
- 表示名が既存の他 MCP サーバーのキーと衝突する場合は接続をブロックし警告する（既存エントリの上書き破壊を防ぐ）。
